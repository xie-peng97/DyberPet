## **DyberPet AI 大脑集成项目 - 技术需求文档**

### 1. 项目概述 (Project Overview)

本项目旨在为桌面宠物应用 `DyberPet` 集成人工智能（AI）功能，赋予宠物一个“大脑”，使其能够与用户进行智能、个性化的交互。通过集成主流的大型语言模型（LLM），如 DeepSeek, ChatGPT, Gemini，将 `DyberPet` 从一个预设行为的动画宠物，转变为一个能理解、能沟通、能记忆、有情感、并能提供实用帮助的智能桌面伴侣。

#### 1.1 项目目标

*   **增强交互性**: 实现宠物与用户的自然语言对话。
*   **塑造个性**: 赋予宠物可爱、粘人、乐观的独特“人格”。
*   **提升智能**: 使宠物能够感知环境、理解指令，并提供助手服务。
*   **建立情感连接**: 通过长期记忆功能，让每个用户的宠物都变得独一无二。
*   **提供用户选择**: 支持多种 AI模型，允许用户自行配置 API Key。
*   **确保隐私安全**: 所有用户数据本地存储，提供明确隐私控制选项。

#### 1.2 目标用户

本项目主要面向**一般电脑用户**，包括学生、职场人士等，他们希望在桌面上拥有一个既有趣又实用的伴侣。

### 2. 功能需求 (Functional Requirements)

以下是本项目需要实现的 AI 功能列表，按优先级和实现阶段划分。

#### 阶段一：核心交互与个性化 (MVP - 最小可行产品)

**FR-1.1: AI 模型配置**

*   **描述**: 在系统的设置界面中，提供一个“AI 设置”面板。
*   **需求**:
    *   提供一个下拉菜单，供用户选择 AI 模型（DeepSeek, ChatGPT, Gemini）。
    *    提供「高级选项」折叠面板（支持模型版本选择）。
    *   提供一个输入框，让用户可以输入并保存他们所选模型的 API Key。
    *    API Key 保存时需进行有效性验证（发送测试请求），失败时阻止保存并提示用户。
    *   API Key 需在本地进行安全存储（使用 AES-256 加密）。
    *    支持本地模型集成扩展（如 Ollama，预留接口）。
    *   提供一个总开关，允许用户一键启用/禁用所有 AI 功能。

**FR-1.2: 基础聊天功能**

*   **描述**: 用户可以通过一个专门的界面与宠物进行文字聊天。
*   **需求**:
    *   在宠物右键菜单中增加“与 DyberPet 聊天”的选项。
    *   点击后，弹出一个包含对话历史记录和用户输入框的聊天窗口，可新建对话，AI自动总结对话标题。
    *    实现回复的**流式输出**（逐字显示）。
    *    回复显示支持markdown格式显示
    *    用户可管理对话（删除/重命名会话）。
    *    用户可中断AI回复生成（添加停止按钮）。
    *   用户发送消息后，宠物能通过所选的 AI 模型生成回复并显示。
    *    定义上下文处理策略：自动截断最旧消息 > 4096 tokens。

**FR-1.3: 宠物人格 (Persona) 注入**

*   **描述**: 所有 AI 的回复都必须符合预设的“宠物人格”。
*   **需求**:
    *   设计一个标准的 **系统提示 (System Prompt)**，定义宠物的身份（DyberPet）、性格（可爱、粘人、乐观）、称呼（称用户为“主人”）、以及说话风格（口语化、带颜文字和emoji表情）。
    *    实现 **Prompt 模板引擎**：动态注入 `{pet_name}`, `{user_name}`, `{mood}`, `{hunger}` 等变量。
    *    将系统提示词移至配置文件 `ai_prompt.yaml` 便于维护。
    *   所有对 AI 模型的 API 请求都必须包含此系统提示。
    *    称用户为“主人”的概率 > 95%，每条回复包含emoji/颜文字。

**FR-1.4: 状态感知对话**

*   **描述**: 宠物的对话内容应能反映其当前的内部状态（饱食度、好感度）。
*   **需求**:
    *   在发往 AI 的请求中，动态地加入宠物的当前状态信息（如 `心情: 开心`, `饱食度: 饿了`）。
    *    注入状态：心情、饱食度、清洁度（根据核心系统变量确定）。
    *    状态变化时实时更新提示词，非活跃对话按每次请求更新。
    *   AI 的回复应根据这些状态信息产生相应变化（例如，饿了的时候可能会提到食物）。

#### 阶段二：主动交互

**FR-2.1: 定时主动问候**

*   **描述**: 宠物能够主动发起对话，而不是一直被动等待。
*   **需求**:
    *   应用内部设置一个定时器（例如，每 30-60 分钟随机触发一次，饿了也会触发一次）。
    *   触发时，AI 会生成一句关心用户的话（如提醒休息、喝水），并通过 `BubbleText` 气泡通知显示。
    *    实现**免打扰模式**：用户可设置不触发时段（23:00-8:00）。
    *    饥饿触发的问候优先级 > 定时问候。
    *    问候语需结合时间/状态生成（30+模板库）。

**FR-2.2: 事件触发对话**

*   **描述**: 在特定事件发生时，宠物能主动交流。
*   **需求**:
    *   **应用启动时**: 宠物会根据当前时间（早/中/晚）进行问候。
    *   **长时间无操作时**: 宠物可以发出一条俏皮的提示，询问用户是否在忙。
    *   「长时间无操作」定义为 **≥45分钟**（用户可配置）。

#### 阶段三：智能助手

**FR-3.1: 待办事项与定时提醒**

*   **描述**: 用户可以通过自然语言让宠物记录待办事项并设置提醒。
*   **需求**:
    *   AI 需要能解析用户的指令，提取出“任务内容”和“提醒时间”。
    *   明确定义Function Calling的JSON Schema（含参数类型/描述）。
    *   参数清洗：自然语言时间统一转换为ISO 8601格式。
    *   错误处理：解析失败时返回友好提示（如“主人，我没听清楚要做什么事/什么时候做呢？”）。
    *   利用 `apscheduler` 设置一个一次性的定时任务。
    *   到达指定时间后，通过 `BubbleText` 气泡通知来提醒用户。
    *   AI 在设置成功后，需要向用户回复一条确认信息。
    *   在 `DashboardUI` 的 `taskInterface` 中，新增一个 `TaskPanel` (待办事项面板)，用于显示、管理和手动完成待办事项。
    *   在`TaskPanel`中显示创建时间/最后修改时间。
    *   待办事项列表需要进行本地持久化存储。
    *   *未来扩展：预留重复任务参数`recurring_rule`*。

**FR-3.2: 简单的知识问答与文本处理**

*   **描述**: 用户可以向宠物询问常识问题或请求简单的文本帮助。
*   **需求**:
    *   在聊天框中，用户可以问问题，AI 以宠物口吻回答。
    *   支持拖拽 `.txt` 文件到聊天框，AI 能对文件内容进行总结。
    *   文件处理：限制.txt文件≤100KB，超限时提示。
    *   显示处理状态（加载动画），失败时返回错误原因。

#### 阶段四：长期记忆

**FR-4.1: 记忆用户关键信息**

*   **描述**: 宠物能记住与用户相关的特定信息，如名字、爱好等。
*   **需求**:
    *   通过后台 AI 任务，从对话历史中自动提取用户的关键信息。
    *   将提取的信息存储在本地的**向量数据库**中。
    *    增加**隐私开关**：默认关闭记忆功能，需用户手动启用。
    *    使用 `all-MiniLM-L6-v2` 本地Embedding模型（平衡性能/精度）。
    *    记忆元数据：存储来源时间、置信度、关联状态变量。

**FR-4.2: 在对话中应用记忆**

*   **描述**: 宠物在后续的对话中，能自然地运用它所记住的信息。
*   **需求**:
    *   在用户发起新对话时，系统会先从向量数据库中检索最相关的记忆片段。
    *   将这些记忆片段作为上下文信息，一同发送给 AI，以生成更具个性化和上下文关联性的回复。
    *    实现记忆管理界面（查看/编辑/删除记忆片段）。
    *    检索策略：最新记忆权重+20%，相关性阈值 > 0.78。

### 3. 技术实现方案 (Technical Implementation Plan)

#### 3.1 核心组件

*   **`AIManager` 类**:
    *   **职责**: 作为所有 AI 功能的统一入口。封装对不同 LLM（DeepSeek, ChatGPT, Gemini）API 的调用方法。
    *   **方法**:
        *   `get_chat_response(prompt, history, model_name, stream_callback: Callable)`: 获取聊天回复（支持流式响应）。
        *   `extract_info_from_text(text)`: 用于提取待办事项或记忆信息。
        *   `get_proactive_message(context)`: 获取主动问候语。
    *   **增强实现**:
        *   `_validate_api_key(key) -> bool`: 实现API Key有效性验证。
        *   `_rate_limiter()`: 请求限速（默认5次/分钟）。
        *   **错误处理策略**:
            * 网络错误：重试2次，间隔3秒
            * 模型错误：切换备用模型（如有配置）
            * 无效响应：返回预设错误消息("主人，我脑袋卡壳了>_<")

*   **AI 配置管理**:
    *   在 `settings.py` 中增加 AI 相关配置项，如 `ai_model`, `api_key`。
    *   使用 `cryptography.fernet` 进行API Key加密存储。
    *   敏感操作日志脱敏（自动过滤API Key）。

#### 3.2 功能调用 (Function Calling)

*   **待办事项提醒 (FR-3.1)** 功能强烈建议使用 **Function Calling / Tool Use** 模式实现。
*   **步骤**:
    1.  向 AI 定义一个名为 `set_todo_reminder` 的本地函数，描述其功能和所需参数 (`task_content`, `reminder_time`)。
    2.  **强化：** 明确定义JSON Schema（参数类型、描述、是否必需）。
    3.  当用户的输入意图与此功能匹配时，AI 会返回一个调用该函数的请求 (JSON 格式)。
    4.  Python 代码捕获此请求，解析参数，并进行**参数清洗**（如自然语言时间转ISO 8601格式）。
    5.  调用 `Scheduler_worker` 中的 `add_todo_reminder` 方法来设置真实的定时任务。
    6.  将函数执行结果（成功/失败）返回给 AI。
    7.  AI 根据执行结果，生成最终的确认消息给用户。
*   **流程图**:
    
    ```mermaid
    sequenceDiagram
        User->>+AI: "下午3点提醒我买咖啡"
        AI-->>-DyberPet: { "function": "set_todo", "args":{ "content":"买咖啡", "time":"15:00"}}
        DyberPet->>+Validator: 清洗时间参数 → "2024-07-10T15:00:00+08:00"
        Validator-->>-Scheduler: 创建任务
        Scheduler->>AI: { "status": "success", "id": 123 }
        AI->>User: "已设置⏰ 下午3点买咖啡！"
    ```

#### 3.3 长期记忆

*   **向量数据库**: 推荐使用轻量级的 `ChromaDB`。
    *   `pip install chromadb`
    *   它无需独立服务器，数据库文件可以直接存储在本地 `data/memory` 目录中。
*   **Embedding 模型**: 使用 `sentence-transformers/all-MiniLM-L6-v2` 本地模型。
*   **工作流**:
    *   **存储**: 对话结束后，异步调用 AI 提取关键信息，并将其 embedding（向量化）后存入 ChromaDB。
    *   **检索**: 新对话开始时，将用户输入 embedding 后，在 ChromaDB 中进行相似度查询，取回最相关的记忆片段。
*   **配置示例**:
    ```python
    client = chromadb.PersistentClient(path="./data/memory")
    collection = client.create_collection(
        name="user_memories",
        metadata={"hnsw:space": "cosine"},
        embedding_function=local_embedder # all-MiniLM-L6-v2
    )
    ```
*   **自动清理策略**: 保留最近100条记忆，超过时删除置信度<0.6的旧记忆。

#### 3.4 数据持久化

*   **待办事项数据**:
    *   创建一个新的 JSON 文件，如 `data/todo_data.json`。
    *   存储 `todo` 和 `done` 两个列表，每个列表中的对象包含 `id`, `content`, `reminder_time`, `created_time`, `last_modified_time` 等字段。
    *   `TaskPanel` UI 的操作（添加、完成、删除）都应实时读写此文件。
*   **对话历史存储**:
    *    使用SQLite数据库存储对话历史（表结构：`session_id, timestamp, role, content`）。
    *   存储路径：`data/chat_history.db`。

### 4. 非功能性需求 (Non-Functional Requirements)

| 类别       | 要求                |
| ---------- | ------------------- |
| **性能**   | 聊天首字响应 ≤1.5s  |
| **稳定性** | AI功能崩溃率 < 0.5% |
| **安全**   | API Key AES-256加密 |
| **隐私**   | 记忆功能默认关闭    |
| **扩展性** | 新AI模型接入 ≤2人日 |

### 5. 测试策略

*   **人格一致性测试**：抽样100条回复，验证称呼/颜文字/语气符合率 ≥90%
*   **Function Calling测试**：覆盖20种自然语言指令变体（解析成功率 >95%）
*   **压力测试**：连续72小时运行，内存泄漏 < 5MB/h
*   **兼容性**：Windows 10/11, macOS Sonoma × Python 3.8-3.11

### 6. 开发阶段与里程碑 (Development Phases & Milestones)

*   **里程碑 1 (v1.0 - AI 核心上线)**:
    *   **目标**: 完成**阶段一**的所有功能。
    *   **交付物**: 用户可以与一个有个性的宠物进行基本对话，并且对话能反映宠物的基本状态。
    *   **验收标准**:
        *   AI 设置界面功能完整（含API Key验证）。
        *   聊天功能可用（支持流式输出）。
        *   宠物的回复符合预设的可爱人设（抽样符合率≥90%）。
        *   [量化] 流式响应延迟 ≤300ms/词。
        *   [安全] API Key 加密存储验证通过。

*   **里程碑 2 (v1.1 - 主动交互)**:
    *   **目标**: 完成**阶段二**的所有功能。
    *   **交付物**: 宠物会主动与用户打招呼和互动。
    *   **验收标准**:
        *   应用启动和定时问候功能正常。
        *   `BubbleText` 能正确显示 AI 生成的内容。
        *   免打扰模式支持配置多个时间段。
        *   主动问候触发准确率 > 98%。

*   **里程碑 3 (v1.2 - 智能助手)**:
    *   **目标**: 完成**阶段三**的所有功能。
    *   **交付物**: 宠物具备待办事项提醒和简单的问答、文本处理能力。
    *   **验收标准**:
        *   Function Calling 流程能正确解析并设置提醒（50次测试失败率<3%）。
        *   `TaskPanel` UI 功能完善，数据能正确存取。
        *   文件处理：支持100次连续拖拽无崩溃。

*   **里程碑 4 (v2.0 - 专属记忆)**:
    *   **目标**: 完成**阶段四**的所有功能。
    *   **交付物**: 宠物具备长期记忆能力，能与用户建立更深的情感连接。
    *   **验收标准**:
        *   `ChromaDB` 集成成功，记忆能够被正确存储和检索。
        *   宠物能在对话中引用之前记住的用户信息（关键信息召回率>85%）。
        *   提供记忆管理界面。
        *   通过第三方隐私审计。
